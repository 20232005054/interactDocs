# 方案生成系统需求文档

## 1. 系统概述

方案生成系统是一个基于 FastAPI 的临床研究方案生成与编辑平台，旨在简化临床研究方案的撰写流程，提供智能化的文档生成、编辑和协作功能。系统采用三栏式布局，包含左侧导航栏、中央交互区和右侧对话栏，支持用户高效完成方案的创建、编辑和管理。

## 2. 功能需求

### 2.1 基础信息录入

| 功能点 | 描述 |
|-------|------|
| 方案标题录入 | 支持填写方案标题，包含字数统计功能，上限 80 字 |
| 研究关键词输入 | 支持输入研究关键词，用于精准定位方案核心研究方向与技术要点 |

### 2.2 标书框架智能生成

| 功能点 | 描述 |
|-------|------|
| 生成方案结构 | 基于录入的基础信息，点击 "生成方案" 按钮，自动生成符合对应项目规范、匹配研究方向的方案结构框架 |

### 2.3 辅助功能支撑

| 功能点 | 描述 |
|-------|------|
| 使用教程 | 提供操作指导，帮助用户快速上手系统 |
| 历史记录 | 追溯过往生成记录，提升使用便捷性 |

### 2.4 核心交互修改界面

#### 2.4.1 左侧导航栏（文档结构树）

| 功能点 | 描述 |
|-------|------|
| 文档结构展示 | 按临床研究方案的标准结构，将文档拆分为项目核心信息、方案摘要、方案全文等一级模块，每个模块下又包含研究目的、入排标准、结局指标等二级子项 |
| 状态标识 | 每个子项右侧有状态标识：✓ 表示已完成，… 表示待完善，方便用户快速定位未完成的内容 |
| 文档名称显示 | 顶部显示当前编辑的文档名称 |

#### 2.4.2 中央交互区（核心操作区）

| 功能点 | 描述 |
|-------|------|
| 内容编辑与预览 | 采用元数据驱动架构，根据左侧选中的章节，自动生成输入文本、选项、Markdown 格式文本或表格，用户可直接编辑 |
| AI 评估 | 调用 AI 对当前章节内容进行合规性、科学性评估，给出修改建议 |
| AI 帮填 | 基于项目元数据和知识库，自动生成对应章节的初稿内容 |
| 回退 | 撤销上一步操作，恢复到之前的版本 |
| 提交 | 确认当前章节的修改，将内容固化到文档中 |
| 快照存档 | 记录不同时间点的文档版本，支持版本回溯 |
| 关联文档 | 显示当前章节所依赖的上级文档（如项目核心信息）和被依赖的下级文档（如方案正文），保证内容一致性 |

#### 2.4.3 右侧对话栏（AI 协作区）

| 功能点 | 描述 |
|-------|------|
| 聊天界面 | 采用聊天界面形式，用户可以直接向 AI 助理发送指令 |
| AI 内容修改 | AI 助理根据用户的提示词，调用接口修改指定区域的内容，并将修改结果返回，用户确认后即可应用到文档中 |
| 模式切换 | 底部有模式切换按钮：Chat 模式 / 修订模式，可以在自由对话和结构化修订之间切换 |

## 3. 非功能需求

| 需求点 | 描述 |
|-------|------|
| 性能要求 | 系统响应时间不超过 2 秒，页面加载时间不超过 3 秒 |
| 安全性 | 实现用户认证和授权，保证数据安全 |
| 可扩展性 | 系统架构应支持后续功能的扩展和集成 |
| 兼容性 | 支持主流浏览器（Chrome、Firefox、Safari、Edge） |
| 可用性 | 系统可用性达到 99.9% |

## 4. 数据需求

### 4.1 核心数据实体

| 实体 | 描述 |
|------|------|
| 方案文档 | 临床研究方案的完整文档信息 |
| 文档章节 | 方案文档的各个章节内容 |
| 文档版本 | 文档的不同版本快照 |
| 用户信息 | 系统用户的基本信息 |
| 操作历史 | 用户的操作记录 |
| AI 评估记录 | AI 对章节内容的评估结果 |

### 4.2 数据流转

1. 用户录入基础信息（方案标题、研究关键词）
2. 系统生成方案结构框架
3. 用户编辑各章节内容，可使用 AI 帮填和 AI 评估功能
4. 系统记录文档版本和操作历史
5. 用户可通过右侧对话栏与 AI 助理交互，修改文档内容
6. 系统保存最终方案文档

## 5. 范围限定

- 系统仅支持临床研究方案的生成与编辑，不涉及其他类型文档
- 系统的 AI 功能基于预设的知识库和规则，不涉及复杂的自然语言处理模型训练
- 系统不提供文档的打印和导出功能，仅支持在线编辑和管理
