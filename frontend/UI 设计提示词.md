## 文档编辑页

```
设计一个专业、清新、现代的科研标书生成工具界面，用于国自然基金等科研项目申报。整体风格严谨不失友好，信息层级清晰，操作引导明确。

**布局结构**：
- 三栏式布局：
  1. 左侧栏：垂直步骤导航（带勾选完成状态的步骤条，如“输入选题信息”“确定标书框架”“生成完整标书”）+ 标书示例预览区（展示PDF样式的标书缩略图）。
  2. 中间主编辑区：顶部显示课题标题、关键词、项目类型；下方为可编辑的标书框架（如“立项依据”“研究背景与意义”等子标题），每个子标题旁配有编辑（铅笔）、删除（垃圾桶）、添加（加号）图标，支持自定义修改、删除、新增子标题；底部为绿色主操作按钮“生成全文”。
  3. 右侧栏：相关文献/相关基金信息展示区，文献条目包含标题、来源、日期、影响因子等，支持分页导航（页码按钮）和条目操作按钮。

**色彩系统**：
- 主色调：清新绿色（用于主操作按钮、选中态边框、功能提示气泡），传递专业、可信的科研氛围。
- 辅助色：浅灰（卡片背景、边框）、白色（页面底色）、深灰（正文文本），少量点缀色（如提示气泡的星星装饰），整体色彩克制，避免干扰核心内容。

**交互与组件**：
- 步骤导航：垂直步骤条，完成步骤显示绿色勾选标记，清晰展示流程进度。
- 编辑控件：子标题旁的编辑/删除/添加图标，点击后触发对应操作，视觉反馈明确。
- 提示气泡：绿色圆角气泡（带星星装饰），用于说明功能（如“子标题支持自定义编辑”），位置不遮挡核心内容。
- 文献列表：条目排版紧凑有序，分页按钮清晰，支持快速翻页查看文献。

**排版规范**：
- 标题层级：主标题加粗，子标题带有序号和缩进，正文文本清晰易读。
- 信息密度：文献列表信息密度适中，关键信息（如影响因子、DOI）突出显示。

**目标场景**：面向科研人员的标书生成工具，需体现严谨性、引导性和高效性，帮助用户快速构建标书框架。
```

### 临床研究方案自动化撰写系统 - 编辑页 UI 设计提示词

```
现在核心编辑页面，整体风格延续科研类工具的「专业严谨+清新友好」调性，以**清新绿色**为主色调，信息层级清晰、操作逻辑闭环，适配临床科研人员的使用习惯。

### 一、整体布局（四分区联动结构）
采用「左侧导航+中间核心编辑+右侧AI交互+底部关联存档」的四分区布局，各区域边界清晰、联动顺畅：
1. **左侧垂直导航栏**：首行是当前文档名字展示`/api/v1/documents/{document_id}`可以查看文档title，点击可以查看最近文档`/api/v1/documents`来获取最近5个文档。文档右侧有新增章节按钮，可以新增父章节，点击新增章节会有小弹窗弹出用以章节名字填写，需要前端自行维护排序索引。文档名字下面是章节展示`/api/v1/documents/{document_id}/chapters`获取各个章节，已完成/编辑中的模块标注绿色对勾标识，点击可以切换是否完成`/api/v1/chapters/{chapter_id}`。支持折叠展开父章节，章节右侧可以点击删除章节按钮`/api/v1/chapters/{chapter_id}`删除当前章节。父章节右侧可以点击新增章节`/api/v1/chapters`来添加子章节，点击新增章节会有小弹窗弹出用以章节名字填写，需要前端自行维护排序索引。
2. **中间核心交互区**：核心区域为**章节文本编辑器**，用于章节文本编辑、MD格式文本编辑，附有md格式预览，下方是核心操作按钮组，ai评估、ai帮填、回退、提交。
3. **右侧AI对话区**：独立竖栏布局，顶部为AI助手头像与欢迎语，中间是气泡式聊天交互区，底部切换「Chat模式/修订模式」切换按钮，支持与AI协作修改方案内容。
4. **底部功能区**：分为「快照存档」「关联文档」两个标签页，标签页头部用绿色高亮；「快照存档」展示历史版本（如“快照20260109”），「关联文档」展示文档依赖关系（项目核心信息-依赖上级、方案正文-结局指标-依赖下级），支持版本回溯与文档关联查看。


### 三、组件设计规范
1. **导航栏组件**：
   - 折叠分组：点击分组标题可展开/收起子模块，展开图标为向下箭头，收起为向右箭头（绿色）；
   - 模块项：hover态为浅绿底色，选中态为深绿底色+白色文字，完成态右侧加绿色对勾（✓）。
2. **核心操作按钮**：
   - 主按钮：「AI评估」「AI帮填」「提交」为**绿色实心按钮**（圆角、加粗文字），「回退」为浅灰底色+绿色文字的弱化按钮，按钮排列紧凑、尺寸统一；
   - 交互反馈：按钮hover时加深绿色饱和度，点击时有轻微下沉动效。
3. **AI对话组件**：
   - 气泡：AI气泡为绿色圆角（左侧带三角），用户气泡为紫色圆角（右侧带三角），气泡内文字左对齐/右对齐对应；
   - 模式标签：「Chat模式/修订模式」为绿色下划线选中态，未选中为浅灰文字。
4. **底部标签页**：
   - 标签头部：绿色底色+白色文字为选中态，浅灰底色+深灰文字为未选中态，圆角设计；
   - 内容区：列表项用浅灰分割线，hover时加浅绿底色，支持点击查看详情。
5. **编辑画布**：
   - 内容生成区：输入框、下拉框、表格的边框为浅绿，聚焦时边框变为深绿；
   - 提示文本：浅灰文字标注“元数据驱动架构”等说明，位置居中。

### 四、交互细节
1. 左侧导航模块点击后，中间核心区实时切换对应编辑内容，同时高亮选中模块；
2. 核心操作按钮点击后，有加载态（绿色进度条/转圈图标），完成后给出绿色提示（如“AI评估完成”）；
3. AI对话区发送消息后，AI实时生成回复，绿色气泡逐步加载，完成后停止；
4. 底部快照存档点击可快速回溯方案版本，关联文档点击可跳转至对应模块编辑页。


### 五、业务适配细节
适配「临床研究方案撰写」专属场景，所有元素需体现医疗科研的严谨性：
- 导航模块严格对应临床研究方案的标准结构（入排标准、结局指标、统计方法等）；
- 编辑画布支持临床专属的表格（如访视安排表、入排标准表），表格样式适配绿色主调；
- AI对话区默认语料贴合临床研究（如“修改主要结局指标”“优化统计方法”），符合科研人员的沟通习惯。
```



## 章节编辑区

```
【核心目标】
设计一款专业医学文档章节编辑区 UI，以“段落为核心单位”，实现层级清晰、操作轻量化、反馈明确的编辑体验，适配 PC 网页端，风格简约专业（浅色系、学术医疗风）。

【整体布局】
1.  三栏布局：左侧目录导航区（固定宽度 20%）+ 中间核心编辑区（70%，可滚动）+ 右侧 AI 助手区（10%，本次聚焦左/中区域）；
2.  配色规范：
    -   主色：#1976D2（深蓝，用于选中态、按钮）
    -   选中态背景：#D4EDDA（浅绿）+ 边框 #1976D2（深蓝）
    -   边框色：#E0E0E0（浅灰）
    -   文字色：#333（正文）、#555（辅助文字）、#999（占位符）
3.  字体规范：
    -   正文：14px 无衬线字体（如思源黑体/微软雅黑）
    -   标题一：20px 加粗
    -   标题二：18px 加粗
    -   标题三：16px 加粗

【左侧目录导航区】
### 视觉规范
-   模块标题：顶部显示“目录”（16px 加粗），右侧配折叠箭头（`<` 图标，14px），背景色 #F8F9FA；
-   目录层级：
    -   一级章节（如“1.研究意义与背景”“2.测试方案”）：左对齐，文字 #555；
    -   二级子章节（如“1.1”“1.2”“2.1”“2.2…”）：缩进 16px，文字 #555；
    -   选中态：对应目录项文字 #1976D2（深蓝），背景 #E8F4F8（浅蓝）；
-   滚动条：目录超出视口时显示，细窄浅灰样式。

### 交互提示
-   点击目录项：页面平滑滚动至中间编辑区对应段落，目标段落编辑框短暂高亮（边框 #1976D2 持续 1s），同时目录项同步高亮；
-   折叠箭头：点击后收起子章节列表，箭头方向切换为 `>`，仅显示一级章节；
-   空状态：无章节时显示“暂无章节，点击新增章节开始编辑”（居中，#999）。

【中间核心编辑区】
### 1. 顶部文本样式工具栏
-   布局：左对齐，前置标签“文本样式”（14px #666），后接 4 个样式按钮：**正文（默认选中）、标题一、标题二、标题三**；
-   按钮规范：
    -   尺寸：80px × 32px，圆角 4px；
    -   未选中：浅灰边框 #E0E0E0，文字 #555；
    -   选中：背景 #1976D2，文字白色；
-   交互提示：
    -   未选中段落时点击样式按钮：轻提示“请先选择要修改样式的段落”（顶部居中，3s 自动消失）；
    -   选中段落后点击：立即切换段落样式并重新渲染。

### 2. 段落编辑容器
-   容器规范：
    -   独立矩形框，边框 #E0E0E0，margin 8px 0，padding 16px；
    -   hover 时边框 #1976D2；
    -   选中时（如图中 1.1 内容区）：背景 #D4EDDA（浅绿）+ 边框 #1976D2（深蓝）；
-   内容区：
    -   段落标题栏：顶部显示段落序号/名称（如“1.研究意义与背景”“1.1”），14px #555，可点击编辑；
    -   文本编辑区：多行输入框，支持 Markdown 渲染（加粗 `**`、斜体 `*`），引用标注 `[1]`/`[2]` 自动以上标展示，placeholder 为“请输入段落内容”（#999）；
-   右侧操作区（垂直排列图标）：
    -   “+”图标：16px，hover 变色 #1976D2；点击后在当前段落下方新增同级段落，默认样式为“正文”，自动聚焦新段落输入框；
    -   垃圾桶图标：16px，hover 变色 #D32F2F；点击弹出确认弹窗（标题“删除确认”，文案“确定删除此段落？删除后无法恢复”，按钮“取消/确定”）。

### 3. AI 辅助按钮
-   布局：编辑区底部居中，两个蓝色按钮：**AI评估、AI帮填**；
-   按钮规范：尺寸 100px × 36px，圆角 4px，背景 #1976D2，文字白色；
-   交互提示：
    -   点击“AI评估”：按钮右侧显示加载动画（转圈图标），评估完成后在段落下方显示评估结果（浅灰背景，14px #666）；
    -   点击“AI帮填”：加载完成后自动填充内容到当前段落，填充内容默认“正文”样式，末尾标注“AI 帮填内容”（浅灰小字）。

【文案规范】
-   操作提示：简洁明确，如“新增段落”“删除段落”“AI 评估中...”；
-   错误提示：友好指引，如“段落内容不能为空”“请先选中段落再修改样式”；
-   状态提示：AI 操作时显示“AI 评估中/AI 帮填中”，完成后显示“评估完成/帮填完成”。

【核心约束】
1.  所有增删/修改操作后，目录与编辑区实时同步（如新增段落，目录立即更新）；
2.  仅支持加粗、斜体两种文本格式，无字体/字号切换；
3.  新增段落默认样式为“正文”；
4.  选中多段时，所有选中段落编辑框背景均为浅绿，右侧 AI 区提示“已选中文本 X 段”。
```
